Description

Your task is to build a simple web application that will allow the creation of music playlists. Songs will be chosen from a list of 193 Beatles songs given in a static data file. As the user chooses songs to be added to the playlist metadata regarding the songs will be displayed on the screen.

Technologies you will need to use:

  • HTML5 for the structure of web pages

  • CSS3 for presentation of web pages

  • JavaScript for client-side programming

Dataset

We will use a dataset that contains metadata about Beatles songs. This dataset is available under a “CC0: Public Domain” license from Kaggle:

https://www.kaggle.com/datasets/chadwambles/allbeatlesspotifysongdata2009remaster

There are 6 CSV files in this dataset, we will use only one of these: TheBeatlesCleaned.csv

You do not need to download this file from Kaggle. It is stored under the data directory on StudRes:

CS2003/Practicals/Practical2-Web1-Playlist/data/TheBeatlesCleaned.csv

Note that the file we provide on StudRes is slightly modified to make parsing easier. Specifically, the lines are separated by \n characters, the columns on a line are separated by a ; (semi-colon) character and the fields are not quoted. See data/readme.txt for more details if you are interested.

This file contains 11 columns and 193 rows (excluding the header line). The column names and their use in this practical are listed in the following.

The first column is “id. The value stored in this column will not be visible on screen, however you may use it as part of your program to identify entries in the dataset.

Columns 2—4 and 11 (“year”, “album”, “song”, “duration_ms”) will be displayed on the screen.

Columns 5—10 (“danceability”, “energy”, “speechiness”, “acousticness”, “liveness”, “valence”) are so called audio features. The Kaggle page for this dataset defines audio features as “how Spotify measures emotion in their music catalog”. You are not required to use these values at all in your implementation, though feel free to use them in creative ways if you want!

Requirements

  • Write a node.js program called convert.js which will convert the CSV file into a JSON file. Choose an appropriate data structure for the JSON file. This program should read data from data/TheBeatlesCleaned.csv and write the output to data/TheBeatlesCleaned.json when run from inside the Web1-playlist directory. Do not use an external CSV parsing library. You may assume that the line separator is “\n” and the column separator is “,” (comma) and that the values do not contain any comma characters in them.

  • Write a web application using HTML, CSS and client side JavaScript with the following features.

  • Create a list of all songs from the data file.

  • Create a placeholder for a playlist. A playlist is a set of selected songs, together with some aggregate information about the contents of the playlist.

  • For each song, design and implement a way of adding it to a playlist.

  • Songs should be listed in the order of addition in the playlist.

  • For every song that was previously added to the playlist, design and implement a way of removing it from the playlist.

  • The playlist should render the list of songs in the playlist. In addition it should show the number of songs in the playlist and the total duration of the playlist.

  • Songs (in the list of all songs and in the playlist) are displayed in the following format:

    • NAME (Album: ALBUM, Year: YEAR, Duration: DURATION)

    • Where NAME comes from the “song” field,
      ALBUM comes from the “album” field,
      YEAR comes from the “year” field, and
      DURATION comes from the “duration_ms” field.

    • Duration is given in milliseconds. Convert it to a MM:SS format (minutes and seconds, separated by a colon). So a value of “120853” (from the song Please Please Me should be displayed as “2:01”). Second values are to be rounded to the nearest integer and zero-padded to two digits.

  • Make sure your program does not depend on the particular dataset you are given. We should be able to add/remove/edit new songs to the CSV file, run convert.js to convert this file to JSON and use your web application with this new JSON file without any further changes. You may assume that the columns in the CSV file and the file format will not change.

  • Test your program fully and document your testing in the report.

We recommend an incremental approach: make sure that each part of your program works before moving on to the next part. Make a safe copy of whatever you have got working so far so that you always have a working solution to submit while you are developing the next part of your solution. This should allow you to experiment with different ways of organising your code and implementing functionality without worrying of breaking things.
